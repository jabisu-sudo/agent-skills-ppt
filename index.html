## Weather Skill
### Steps
1. Parse city name from user input
2. Call: curl -s "wttr.in/{city}?format=j1"
3. Format: temp, humidity, wind, forecast
4. Fallback: Open-Meteo API if wttr.in fails

### Output Format
- Current: 🌡️ {temp}°C, {condition}
- Tomorrow: {high}/{low}°C

Description = Routing Signal

"Not documentation for humans — a routing signal for AI."

❌ Vague

"A weather tool"

✅ Precise

"Get weather and forecasts via wttr.in.
Use when: user asks about weather, temperature, or forecasts.
NOT for: historical data, severe weather alerts."

3rd person · Trigger conditions · "NOT for" boundaries

Constraint Spectrum

🧱 High Constraint

curl -s "wttr.in/{city}?format=3"
Do NOT use any other weather API.

🌄 Low Constraint

Use wttr.in or Open-Meteo.
Choose format based on user needs.

AI keeps getting it wrong → tighten. Too rigid → loosen.

Test Your Triggers

⚡ Under-trigger

"苏州明天热不热" → skill not invoked

Fix: add "hot", "cold", "temperature" to description

🚫 Over-trigger

"I feel cold inside" → weather skill fires

Fix: add "NOT for: emotional expressions, metaphors"

Structural Tips

📏 Keep It Short

SKILL.md under 500 lines. Longer → more AI errors.

🚫 Avoid Deep Nesting

AI loses track after 3+ levels of if-else.

✅ Use Numbered Steps

Checklists > paragraphs. Harder to skip steps.

We've Seen This Before

"Agent Skills in 2026 = npm in 2016"

Same growth curve. Same security lag. Same opportunity.

📦 npm ~2016

  • No lockfile — versions drifted silently
  • No publish 2FA — accounts hijacked easily
  • No security audit — npm audit didn't exist yet

🧠 Skills 2026

  • No lockfile — no version pinning yet
  • No publish 2FA — anyone can push updates
  • No security audit — no skill audit equivalent

Enterprise Defense

🔒

Least Privilege

Allowlist skills. Don't give full filesystem access.

🧑‍💻

Human-in-the-Loop

Critical ops need human confirmation.

📦

Sandbox Isolation

Run scripts in sandboxed environments.

📊

Audit Logging

Log every skill invocation for forensics.

Skills vs MCP vs Function Calling

"MCP Will Shrink"

"MCP services will converge to truly complex connections."

🧠 Baked Into Models

Search, code execution, image understanding — already built-in. More capabilities will follow.

📜 Simple Tools → Scripts

Check weather? Calculate something? A bash script is enough — no MCP server needed.

⌨️ Direct CLI Calls

gh, git, curl — already have CLIs. AI calls them directly, no wrapper layer needed.

Challenges Ahead

📈

Context Bloat

100+ skills → routing index alone grows large. Hierarchical routing or dynamic pruning needed.

⚔️

Skill Conflicts

Two skills with similar descriptions → AI picks wrong one. Or two skills give contradictory instructions.

🔄

Staleness

Skills are static files, but APIs change, best practices evolve. No auto-update mechanism yet.
