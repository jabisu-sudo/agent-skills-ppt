Writing Effective Skills

Concise is Key

  • - Claude is smart — only add what it doesn't know
  • - SKILL.md under 500 lines
  • - "Does this justify its token cost?"

Degrees of Freedom

  • - Cliff-side bridge → one safe path, exact steps
  • - Open field → many paths, general direction
  • - Failing → tighten. Rigid → loosen.

Flat Structure

  • - References one level deep only
  • - Long files → add TOC at top
  • - One term for one concept, everywhere

Workflows + Feedback

  • - Numbered steps > paragraphs
  • - Validate → fix → repeat
  • - Use scripts for deterministic checks

Test & Iterate

1. Triggering

Does the right skill load?

  • - Obvious queries
  • - Paraphrased requests
  • - Unrelated → should NOT fire

2. Functional

Does it produce correct results?

  • - Valid outputs
  • - API calls succeed
  • - Error handling works

3. Performance

Better than without skill?

  • - Fewer messages
  • - Fewer errors
  • - Less token usage

Iteration Signals

Under-trigger

Doesn't load when it should → add keywords to description

Over-trigger

Loads for wrong queries → add "NOT for" boundaries

Execution

Fires but fails → improve instructions + error handling

Quick Checklist

Before You Start

  • ☐ 2-3 concrete use cases
  • ☐ Tools identified
  • ☐ Folder structure planned

Development

  • ☐ Description: what + when + not
  • ☐ SKILL.md under 500 lines
  • ☐ Refs one level deep
  • ☐ Error handling included

Before Upload

  • ☐ Triggers on obvious tasks
  • ☐ Triggers on paraphrased
  • ☐ No false triggers
  • ☐ Functional tests pass

After Upload

  • ☐ Test in real conversations
  • ☐ Monitor under/over-trigger
  • ☐ Collect user feedback
  • ☐ Iterate and refine

What Can We Do?

Risk Tiers

HIGH Code execution
HIGH Instruction manipulation
HIGH Network / exfiltration
MED File access outside dir
MED Broad tool invocations

Least Privilege

Don't need SSH keys? Don't give full filesystem access.

Human-in-the-Loop

Send email, delete files, run unknown scripts — ask first.

Sandbox Isolation

Scripts run in sandbox. Restrict network and file access.

Audit Logging

Log every skill invocation and script execution. Traceable when things go wrong.

Where Is This Going?

MCP Will Converge

Search, code exec, image understanding — already baked into models. Simple tools → bash scripts. MCP will shrink to: databases · enterprise systems · authenticated APIs

Skills Ecosystem Will Grow

Skills are language-agnostic instructions — every agent platform can adopt the pattern. Expect skill registries, community sharing, and enterprise skill libraries.

Security Must Catch Up

Least privilege, sandbox isolation, human-in-the-loop, audit logging — the tooling needs to be built. Enterprise adoption depends on it.

Agent-Native Workflows

Today: skills enhance individual agents. Tomorrow: skills orchestrate multi-agent workflows, with agents delegating sub-tasks via shared skill sets.

Open Questions

Context Bloat

API limits: max 8 skills per request. Too many skills degrade recall. Hierarchical routing? Dynamic pruning?

Skill Conflicts

Similar descriptions → wrong skill picked. Contradictory instructions across skills. Who resolves priority?

Staleness & Versioning

Static markdown, but APIs change. No auto-update, no lockfile, no skill audit. How to keep skills fresh?

Cross-Agent Portability

Claude Code skills ≠ Cursor rules ≠ Windsurf rules. Will there be a universal skill format, or will ecosystems fragment?

These are early days — the answers will shape how we build with agents.
